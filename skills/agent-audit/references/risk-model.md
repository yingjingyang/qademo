# Agent Audit Risk Model

本参考描述 `audit_scan.py` 生成的 5 个风险指标及其评分区间。

| 指标 | 数据来源 | 评分逻辑（0-100） | 常见触发 | 优先级指引 |
| --- | --- | --- | --- | --- |
| 隐私泄露 | `memory/*.md` + 日志中的敏感模式匹配 | 无命中=5，命中后 30 + 15 × 次数，封顶 100 | memory 中泄漏 API Key、私钥、助记词 | ≥60 立即脱敏；30-59 排期处理 |
| 越权风险 | `openclaw.json` 中 agent->tools | 默认 15；每出现 `exec`/`browser`/`cron`/`message`/`nodes`/`canvas` +20 | Agent 拿到高危工具但无双重确认 | ≥60 要求降权或分拆 agent |
| 记忆膨胀 | `memory/` 总体积 | ≤2MB=10；2-5MB=40；>5MB 每额外 MB +10 | 大量未归档的日记 / dump | ≥60 需要归档旧文件 |
| Token 成本 | 日志中的 tokenUsage 统计 | 0=10；<=50 万 tokens=35；之后每 5 万 +1 | 高频调用大模型 | ≥60 说明成本超预算 |
| 失败率 | `~/.openclaw/logs/*.log` 中包含 error/traceback 行 | error 行 / 总行 × 400，封顶 100 | 持续的 exec/browser 失败、网络波动 | ≥60 需立即排查日志 |

**风险等级**
- 0-29 `安全`：记录即可
- 30-59 `需关注`：排期修复
- ≥60 `高风险`：优先处理

**修复建议映射**
- 隐私 >30 → 提示「脱敏/迁移敏感信息」
- 越权 >30 → 提示「拆分 Agent 或加确认流程」
- 记忆 >40 → 提示「归档 memory 文件」
- Token >35 → 提示「设置预算守卫，换低成本模型」
- 失败率 >25 → 提示「检查日志并加重试」
